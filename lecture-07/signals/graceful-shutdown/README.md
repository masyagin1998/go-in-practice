# graceful-shutdown

Базовый паттерн: SIGINT/SIGTERM → cancel контекста → горутины доделывают
текущие задачи и выходят.

## Что здесь важно

- **`signal.NotifyContext`** (Go 1.16+) — самый компактный способ
  превратить подписку на сигнал в контекст. Раньше писали
  `signal.Notify + <-ch + cancel()`, теперь — одна строка.
- **Контекст прокидывается всюду**: воркер не крутит `time.Sleep(X)`,
  а делает `select { case <-ctx.Done(): …; case <-time.After(X): … }`.
  Без этого shutdown ждёт самого долгого `Sleep`.
- **`sync.WaitGroup` + `wg.Wait()`** — единственный правильный способ
  убедиться, что все горутины закончили. `os.Exit` без `wg.Wait()` режет
  их посередине.

## Ручной запуск

```bash
go run .
# в другом терминале (или Ctrl+C):
kill -TERM $(pgrep -f graceful-shutdown)
```

Будет видно: продюсер перестал кидать, каждый воркер допечатывает
последний обработанный item, потом общее "выходим".

## Частые ошибки

- **Небуферизованный канал сигналов**. Если канал ёмкости 0 и никто
  прямо сейчас не читает — сигнал потеряется. `signal.NotifyContext`
  делает ёмкость 1 автоматически, для ручного `signal.Notify` —
  ставьте `make(chan os.Signal, 1)` как минимум.
- **Обработка SIGKILL**. Нельзя. `SIGKILL` и `SIGSTOP` доставляются
  ядром напрямую, процесс не успевает ничего сделать. Отсюда правило:
  всё, что важно сохранить — пишите на диск с fsync **до** SIGTERM,
  а не в shutdown-обработчике.
