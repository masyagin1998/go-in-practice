# thread-masking — сигналы и маски тредов

`kill(pid, signo)` — **process-directed** сигнал. Ядро выбирает
ЛЮБОЙ тред процесса, у которого `signo` не заблокирован в
`pthread_sigmask`. Если таких тредов несколько — распределение
произвольное (на Linux обычно round-robin).

Этот пример показывает, что **маска живёт на уровне треда**, а не
процесса. Четыре goroutine'ы пинятся через `runtime.LockOSThread`,
первые две оставляют SIGUSR1 заблокированным, последние две —
разблокируют. Peer шлёт 100 SIGUSR1. Итог: только треды 2 и 3 ловят.

## Запуск

```bash
make peer
# терминал 1:
go run .
# терминал 2 (подставьте PID из первого):
./peer <PID> 100
# Ctrl+C в первом терминале для итогов.
```

Или `make run` — всё автоматизирует.

## Почему блокировка в C-конструкторе

Чтобы Go runtime-треды (GC, scheduler, sysmon и т.д.) не оттянули
на себя сигнал, блокируем SIGUSR1 ДО старта runtime'а. Конструктор
`__attribute__((constructor))` выполняется до `runtime·rt0_go`, главный
тред стартует с блокировкой, все Go M-треды наследуют маску через
`clone()`. Go при создании новых M не трогает "blockable" сигналы
(SIGUSR1/2 такие), так что маска сохраняется.

Без этой предосторожности kernel мог бы доставить сигнал runtime-треду,
который при отсутствии пользовательского обработчика для SIGUSR1
**завершит программу** (default action). Если бы был `signal.Notify`,
сигнал шёл бы в Go-канал — но ssi_int/ssi_pid мы бы не увидели.

## Связь с `signalfd`

Соседний пример `../signalfd/` использует эту же идею — блокировать
сигналы на всех тредах, — но вместо sigaction-хендлера ставит `signalfd(2)`
и читает `struct signalfd_siginfo` из fd. signalfd и handler-on-thread —
два альтернативных способа приёма сигналов в многотредных процессах.

## Что увидите

```
Результат:
  thread 0 (✗ заблокирован): поймал 0 сигналов
  thread 1 (✗ заблокирован): поймал 0 сигналов
  thread 2 (✓ разблокирован): поймал 53 сигналов
  thread 3 (✓ разблокирован): поймал 47 сигналов
```

Сумма 2+3 ≈ 100 (все отправленные сигналы доставлены), распределение
между ними — что ядро выбрало.

## А если НЕ блокировать?

Уберите `__attribute__((constructor))` и не делайте `unblock`. Тогда
все треды не блокируют SIGUSR1 → ядро доставляет любому. Один из
Go-runtime тредов получит сигнал, default action для SIGUSR1 = terminate.
Программа молча умрёт. Попробуйте — наглядно.
