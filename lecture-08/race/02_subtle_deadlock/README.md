# 02_subtle_deadlock — рантайм НЕ ловит, `-race` НЕ ловит

Те же AB-BA-мьютексы, что и в 01, но `main` живёт своей жизнью (тикер).
Рантайм видит "main runnable / просыпается по таймеру → программа жива"
и `checkdead()` молчит. `-race` тоже бесполезен: data race нет, есть
просто два повисших `Lock()`.

## Запуск

```bash
go run .          # тикер тикает, программа выходит чисто
go run -race .    # то же самое, ничего диагностического
```

## Чем такое ловить

- `go.uber.org/goleak` — упавший тест в конце жизни программы:
  "после `main` остались живые goroutine — вот эти". Демо в `goleak/`.
- `runtime/pprof` (`/debug/pprof/goroutine?debug=2`) — снимок стеков
  всех goroutine, повисшие сразу видны.
- `GODEBUG=schedtrace=1000` — в трейсе шедулера видны заблокированные G.

## Мораль

`-race` ≠ "детектор всех проблем конкурентности". Он строго про data race
(unsynchronized read+write одной памяти). Дедлоки, leak'и goroutine,
голодание, неверный happens-before на бизнес-логике — отдельные тулы.
