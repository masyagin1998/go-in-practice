# 04_runtime_stack — дамп стеков без дебагера

Два способа получить стеки всех goroutine из работающего процесса —
без `dlv`, `gdb` и перекомпиляции.

## Способ 1: SIGUSR1 + runtime.Stack

Программа ловит `SIGUSR1` и печатает в stderr вывод `runtime.Stack(buf, true)`.
Процесс продолжает работу.

```bash
make run            # программа крутится, через 1 сек получает SIGUSR1
# либо вручную:
go run . &
kill -USR1 $!
```

Артефакт `stacks.txt` — реальный вывод дампа.

## Способ 2: GOTRACEBACK=all + SIGQUIT

Без изменения кода. В Go-рантайме есть встроенный обработчик: SIGQUIT
(Ctrl-\ в терминале) → печать стеков всех goroutine → завершение.
Переменная `GOTRACEBACK=all` управляет объёмом:

| Значение | Что печатает |
|---|---|
| `none` | только причину краша |
| `single` (default) | стек падающей goroutine |
| `all` | стеки всех пользовательских goroutine |
| `system` | + goroutine рантайма |
| `crash` | + core dump после |

```bash
GOTRACEBACK=all go run . &
kill -QUIT $!          # процесс умрёт с дампом
```

Минус способа 2: процесс погибает. Плюс — работает буквально на любом
Go-бинарнике без единого изменения.
